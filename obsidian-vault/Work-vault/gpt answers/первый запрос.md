Да, по этим файлам уже можно понять значительную часть математики, физики и pipeline приложения. Но не всю. Самая важная часть обработки сырых ультразвуковых данных видна в пакете `m2`, в классах `d`, `e`, `r`, `s`, `z`, а также частично в нативной библиотеке `pvus.so`. Java код показывает геометрию, scan conversion, отрисовку B mode, Color Doppler, Power Doppler, PW Doppler, M mode, needle navigation, B line detection и сохранение кадров. Нативный код показывает алгоритмы анализа PW кривой и анализа направления сосуда.

Ниже разберём по слоям.

## 1. Общая картина pipeline

Приложение работает как портативный ультразвуковой сканер. Данные приходят от датчика, складываются в объект `m2.d0`, дальше проходят scan conversion, цветовое кодирование, наложение режимов и отрисовку.

Упрощённый pipeline выглядит так:

```text
Датчик
  ↓
Сырые данные кадра
  ↓
m2.d0
  ↓
b_data, color_data, power_data, pw_data
  ↓
scan conversion
  ↓
Bitmap B mode
  ↓
Color или Power overlay
  ↓
PW или M mode overlay
  ↓
USImageView или USImageSurfaceView
  ↓
Canvas
```

Ключевой контейнер кадра:

```java
public class d0 {
    public Date f8788e;
    public Map<String, Object> f8789f = new HashMap();
}
```

Внутри `f8789f` лежат разные типы данных:

```text
"b_data"       → m2.u      B mode данные
"color_data"   → m2.y      Color Doppler данные
"power_data"   → m2.c0     Power Doppler данные
"pw_data"      → m2.b0     Pulsed Wave Doppler данные
"miscs"        → разные дополнительные данные, например needle_data
```

## 3. Scan conversion

Самая важная математика находится в `m2.d`, `m2.e`, `m2.r`, `m2.z`.

Scan conversion это преобразование из ультразвуковой координатной системы в экранную декартову.

Исходные данные B mode обычно лежат в полярной или секторной сетке:

```text
x исходный луч
y глубина вдоль луча
```

Для отображения на экране нужно получить:

```text
screenX
screenY
```

### 3.1 Геометрия датчика

В `m2.o`:


Здесь есть два типа геометрии.

Первый тип:

```java

```

Второй тип для линейного датчика делает фиктивный огромный радиус:

```java

```

Это классический трюк: линейный датчик можно представить как сектор с очень большим радиусом.

### 3.2 Параметры scan conversion

Это важная физическая константа. `770000` похоже на половину скорости звука в тканях в единицах, согласованных с частотой дискретизации:

```text

половина скорости около 770000 мм/с
```

В ультразвуке расстояние считается по времени туда и обратно:

```text

```

Поэтому появляется половина скорости звука.


В коде это `f8804k`.

## 4. Прямая и обратная координатная математика

Класс `m2.r` хранит готовую геометрию отображения. Именно он отвечает за функции:

```java
a(PointF pointF)      // raw coordinate → display coordinate
A(PointF pointF)      // display coordinate → raw coordinate
J(PointF pointF, angle) // raw coordinate с поворотом steer
B(PointF pointF, angle) // display coordinate назад с учётом steer
```

### 4.1 Исходная координата в физическую

Метод `r.a`:

```java
double d4 = pointF.x;
double d5 = pointF.y * this.f8861p;
float radius = this.f8858m;
double r = d5 + radius;
```

То есть:

```text
beamIndex = x
sampleIndex = y
physicalDepth = y * spacing
radialDistance = radius + physicalDepth
```

Если радиус отрицательный, используется инвертированная геометрия.

Угол луча:

```java
double angle = sectorAngle / (lineCount - 1) * (beamIndex - (lineCount - 1) / 2)
```

Дальше полярные координаты переводятся в декартовы:

```java
screenX = sin(angle) * radialDistance / pixelSpacing + originX
screenY = cos(angle) * radialDistance / pixelSpacing + originY
```

В коде:

```java
double dSin = Math.sin(d8) * d6;
double dCos = d6 * Math.cos(d8);

pointF2.x = dSin / f8866u + origin.x;
pointF2.y = dCos / f8866u + origin.y;
```

Затем вызывается `M`, которая масштабирует под фактический размер View.

### 4.2 Обратное преобразование

Метод `r.A` делает обратное:

```java
PointF pointFL = L(pointF);
float dx = pointFL.x - origin.x;
float dy = pointFL.y - origin.y;

double physicalX = dx * pixelSpacing;
double physicalY = dy * pixelSpacing;

double radiusDistance = sqrt(physicalX² + physicalY²);
double angle = atan2(physicalY, physicalX);
```

Затем восстанавливает:

```text
sampleIndex = (radiusDistance - radius) / spacing
beamIndex = angle → индекс луча
```

Это нужно для тачей, ROI, Doppler box, needle overlay и измерений.

## 6. Grayscale LUT и динамический диапазон

Класс `m2.h` содержит LUT.

Пример gamma LUT:

```java
for (int i = 0; i < 256; i++) {
    value = pow(i / 255.0 - 0.05, gamma)
    value = clamp(value, 0, 1)
    lut = value * 65280
}
```

Фактически это преобразование яркости:

```text
output = clamp((input / 255 - 0.05)^gamma, 0, 1)
```

Где gamma ограничена:

```java
if (gamma < 0.1) gamma = 0.1;
if (gamma > 4.0) gamma = 4.0;
```

Есть ещё sigmoid LUT:

```java
dArr2 = d4 + ((d5 - d4) / (exp((-d7 * 4.0) * (x - d6)) + 1.0));
```

Это S образная кривая контраста:

```text
output = low + (high - low) / (exp(-k * (x - center)) + 1)
```

Такие LUT обычно используются для динамического диапазона, контраста и enhancement.

В `setPwValue` есть ключевая формула скорости:

```java

    (((this.C.f8719j * 1000000.0f) * 4.0f)
```

```java

    (((this.F.f8719j * 1000000.0f) * 4.0f)
```

Это шкала скорости для спектрального Doppler.

Физика Doppler:

```text
fd = 2 * f0 * v * cos(theta) / c
```

Отсюда:

```text
v = fd * c / (2 * f0 * cos(theta))
```

В коде есть похожая структура:

```text
velocityScale ∝ PRF * c / (f0 * cos(theta))
```

Здесь константа `154000.0f` вероятно в см/с или в другой нормированной системе, связанной с `1540 m/s`.

`f8719j * 1000000` это частота в Hz, если `f8719j` хранится в MHz.

Почему делитель `4.0f`:

```text
может быть связано с FFT bin scaling, IQ demodulation, Nyquist range или внутренней нормировкой спектра
```

Без полного native кода точную интерпретацию множителя 4 утверждать нельзя.

## 16. Needle navigation

Данные иглы приходят в `MainActivity.l2`:

```java
for (int i = 0; i < 192; i += 4) {
```

То есть приходят 48 чисел по 4 байта:


Каждое делится на 4. Вероятно это fixed point координаты или магнитные параметры.


## 17. Vessel direction analysis из pvus.so

В нативном коде виден класс `VesselDirAna`.

Основные методы:

```cpp
VesselDirAna::setROI(unsigned char*, int width, int height, int param4, int param5)
VesselDirAna::calc()
VesselDirAna::getResult(float&, int&, int&, int&, int&)
```

### 17.1 setROI

```cpp
param_2 = width
param_3 = height
```

## 18. PW curve analysis из pvus.so

Класс `VFCalcCore` анализирует спектр PW.

Методы:

```cpp
SetRawData(unsigned char*, unsigned int size, unsigned int baseline)
GenCurve(int&)
FindPeak()
FindSD()
```

### 18.1 SetRawData

```cpp
this + 0x18 = size
this + 0x1c = baseline
this + 0x08 = raw copy
this + 0x10 = working copy
this + 0x20 = curve int array
this + 0x28 = peak result
this + 0x30 = SD result
```

Размер curve:

```cpp
param_2 >> 5 & 0x7fffffc
```

Если кадр PW имеет высоту 128, то `size >> 7` часто используется как количество столбцов.

В `GenCurve` видно:

```cpp
uVar7 = param_1 >> 7;
```

То есть:

```text
param_1 = total pixels
width = totalPixels / 128
height = 128
```

PW spectrum анализируется как изображение высотой 128 пикселей.

### 18.2 GenCurve pipeline

Начало:

```cpp
medianFilter(3,5)
meanFilter(5,5)
dilation(5,5)
erosion(5,5)
```

Это обработка изображения спектра:

```text
median filter      убирает соль и перец
mean filter        сглаживает шум
dilation           расширяет яркие области
erosion            сужает обратно
```

Dilation + erosion это морфологическое closing или opening в зависимости от порядка. Здесь:

```text
dilation затем erosion
```

Это closing: закрывает разрывы в ярком спектре.

Дальше:

```cpp
fuzzyCMean(..., 0.6)
fuzzyCMean(..., 0.2)
```

Получаются два порога:

```text
iVar26 основной threshold
iVar27 более мягкий threshold
```

Дальше код считает яркие пиксели выше порога в положительной и отрицательной областях относительно baseline:

```cpp
POS DOT
NEG DOT
POS AREA
NEG AREA
```

На основании этого решает:

```text
валидная ли PW wave
направление потока
искать вверх или вниз относительно baseline
```

Если сигнал невалидный:

```cpp
INVALID PW WAVE
return 0
```

### 18.3 Построение кривой

Дальше создаются две бинарные маски:

```cpp
puVar32 = pixels > threshold1
__s     = pixels > threshold2
```

Для каждого столбца ищется длинный сегмент ярких пикселей в нужном направлении от baseline. Результат пишется в curve array:

```cpp
param_1 = curve
```

То есть `GenCurve` строит envelope curve спектра:

```text
для каждого x времени:
    найти y максимальной скорости
```

В конце:

```cpp
curveMedianFilter(5)
затем moving average 5 точек
```

Финальная кривая сглаживается.

### 18.4 FindPeak

`FindPeak` ищет пики на curve.

Берёт:

```cpp
baseline = *(int *)(this + 0x1c)
curve = *(int **)(this + 0x20)
width = size >> 7
```

Считает амплитуду:

```cpp
abs(curve - baseline)
```

Ищет наиболее сильные пики, группируя соседние точки. В конце, если найдено больше двух пиков:

```cpp
result = peak
result = peak
result = peak
```

Иначе:

```cpp
result = {-1, -1, -1}
```

То есть возвращает последние три значимые peak position по времени.

### 18.5 FindSD

`FindSD` ищет systolic или diastolic point рядом с peak.

Код:

```cpp
iVar4 = peak - 20;
```

Потом двигается назад и смотрит наклон кривой:

```cpp
slope = (prevValue - currentValue) / deltaX
```

Останавливается, если:

```text
плато длиннее 9 пикселей
или наклон сменил знак
или вышли за 159 пикселей назад
```

Возвращает:

```cpp
sd = peak
sd = foundIndex
```

Похоже, это поиск точки диастолы или начала систолического участка на Doppler envelope.

## 19. Что именно видно по физике

Можно уверенно сказать:

### 19.1 Используется скорость звука около 1540 м/с

В scan conversion:

```java
770000 / sampleRate
```

Это половина скорости звука.

В PW шкале:

```java
154000
```

Это скорость звука в см/с:

```text
1540 м/с = 154000 см/с
```

### 19.2 Глубина считается по time of flight

Формула:

```text
depth = c * t / 2
```

В коде это замаскировано через spacing:

```text
sampleSpacing = 770000 / samplingFrequency
```

### 19.3 Doppler velocity correction

Формула в коде:

```text
velocityScale ∝ c / (f0 * cos(theta))
```

Это соответствует Doppler equation:

```text
v = fd * c / (2 * f0 * cos(theta))
```

### 19.4 Sector scan geometry

Для convex или phased датчика:

```text
x = r * sin(theta)
y = r * cos(theta)
```

Для linear датчика используется большой радиус:

```text
radius = 100000
```

Это приближает сектор к прямоугольной геометрии.
